# МосГорБюджет.Трек

Интерактивный образовательный проект о бюджете Москвы на 2026 год. Версия подготовлена для второго тура Всероссийского конкурса «Бюджет для граждан» в номинации «Бюджет и технологии будущего».

Ключевая гипотеза проекта — ежедневная гражданская привычка: житель изучает один новый фрагмент бюджета, принимает решение в симуляции, видит измеримый прирост знаний и получает подготовленный к передаче городской результат. В перспективе этот результат может быть связан с «Активным гражданином» и «Миллионом призов» через Mos ID.

Сайт: https://moscow-budget-dpbm.vercel.app/

> Это конкурсный прототип, а не официальный сервис Правительства Москвы. Боевой OAuth/API-контур Mos ID и реальные начисления «Миллиона призов» здесь не подключены: интерфейс показывает честный пилотный сценарий, журнал идемпотентности и место будущего серверного обмена, не выдавая его за действующую интеграцию.

## Задача проекта

Бюджетные таблицы сложны для первого знакомства. Проект даёт жителю понятную точку входа: показывает ключевые параметры бюджета, предлагает учебные сценарии, объясняет налоговый вычет и закрепляет материал через задания. Ежедневный квиз возвращает пользователя в продукт, а входной и итоговый тесты превращают вовлечённость в измеримый образовательный результат.

## Что реализовано

- адаптивный интерфейс для компьютеров и смартфонов, светлая и тёмная темы;
- вводная экскурсия с маскотом Фини;
- учебный калькулятор социального налогового вычета со ссылкой на ФНС;
- пять тематических викторин и проверенный банк вопросов;
- пять мини-игр и симулятор распределения условного районного бюджета;
- интерактивная структура расходов и режим сравнения масштаба вычета;
- локальный справочник по заранее подготовленным темам;
- учебные баллы, уровни и прогресс, хранящиеся только в браузере.
- единый реестр бюджетных фактов с периодом, статусом доказательности и прямой ссылкой на источник;
- входная и итоговая диагностика из пяти сопоставимых компетенций с расчётом прироста в процентных пунктах;
- ежедневный квиз с фиксированной подборкой на день, серией возвращений и отдельным журналом баллов-кандидатов городского пилота;
- интерактивная схема Mos ID → проверка результата → «Миллион призов», явно отмеченная как контур, требующий партнёрского API.

## Граница достоверности

Верхнеуровневые показатели 2026 года основаны на официальных источниках:

- доходы — 5 937,4 млрд ₽;
- расходы — 6 385,0 млрд ₽;
- дефицит — 447,6 млрд ₽;
- социальная сфера — около 3,2 трлн ₽;
- образование — 814,6 млрд ₽;
- социальная поддержка — 810 млрд ₽;
- здравоохранение — 615 млрд ₽.

Доли подкатегорий, районные суммы, индексы и игровые результаты являются демонстрационными. Это явно отмечено в интерфейсе. Калькулятор показывает ориентир по базовой ставке 13%; фактический возврат зависит от налоговой базы, ставки и уплаченного НДФЛ.

Баллы в шапке — учебные и локальные. Баллы-кандидаты ежедневного задания — отдельный пилотный журнал: в демонстрации они не уходят во внешнюю систему и не превращаются в реальные промокоды или призы.

## Технологии

React 19, TypeScript, Vite, Tailwind CSS, Motion и lucide-react. Приложение полностью клиентское: у него нет бэкенда, внешней авторизации и API-ключей. Пользовательские настройки сохраняются в `localStorage`.

## Локальный запуск

Требуется Node.js 20.19 или новее.

```bash
npm ci
npm run dev
```

Проверка перед публикацией:

```bash
npm run lint
npm run build
npm audit
```

Production-сборка создаётся в `dist/` как один HTML-файл.

## Следующий серверный контур

После согласования с владельцами систем потребуется: зарегистрировать OAuth/OIDC-клиент Mos ID; принимать только серверный callback с устойчивым идентификатором пользователя; подписывать результат задания и хранить журнал идемпотентности; передавать подтверждённое событие оператору «Активного гражданина»/«Миллиона призов»; вести аудит, согласия и отзыв доступа. Текущая версия демонстрирует пользовательский сценарий без имитации действующей интеграции.

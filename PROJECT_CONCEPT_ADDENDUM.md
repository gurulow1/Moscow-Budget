# Дополнение к концепции «МосГорБюджет.Трек»

Конкретный план внедрения, пилотные метрики и чек-лист конкурсной подачи вынесены в отдельный файл: [«Предложения по реализации и практическому применению»](./ПРЕДЛОЖЕНИЯ_ПО_РЕАЛИЗАЦИИ_И_ПРАКТИЧЕСКОМУ_ПРИМЕНЕНИЮ.md).

## Ежедневная гражданская петля

Проект — не каталог мини-игр, а ежедневный маршрут: житель изучает один новый фрагмент бюджета, применяет знание в калькуляторе/квизе/сценарии «Виртуальный мэр», видит измеримый результат и возвращается за новым заданием. После партнёрского подключения подтверждённое действие может передаваться через Mos ID в контуры «Активного гражданина» и «Миллиона призов».

В текущем конкурсном прототипе реальные городские баллы не начисляются. Интерфейс показывает отдельный журнал «баллов-кандидатов», анти-повтор на сутки и границу будущего серверного обмена — без выдачи макета за боевую интеграцию.

## Измеримый результат

До практики пользователь проходит пять входных вопросов. После сохранения расчёта и одной миссии открывается сопоставимый итоговый тест. Система показывает входной и итоговый процент, прирост в процентных пунктах и локальное время прохождения. Это позволяет оценивать не только вовлечённость, но и learning gain.

## Источники и данные

Единый реестр находится в `src/data/budgetFacts.ts`. Каждый факт содержит значение в млрд рублей, период, статус (`official`, `derived` или `educational-model`), источник и даты публикации/проверки. Аналитика использует официальные рублёвые суммы напрямую; округлённые доли не используются для обратного вычисления денег.

## Пилотный серверный контракт

```text
Mos ID (OAuth/OIDC)
  → устойчивый city user id
  → подписанный результат задания + антифрод
  → идемпотентный журнал оператора
  → «Активный гражданин» / «Миллион призов»
```

Для production понадобятся согласование с владельцами систем, OAuth-клиент, серверная проверка подписи, один зачёт в сутки, аудит, отзыв доступа и договорённый API начисления.
